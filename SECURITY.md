# Security Policy

Report security issues to the maintainers promptly. Do not open public issues for sensitive vulnerabilities — instead contact the project maintainers via email (hello@yachi.app) or create a private report if your platform supports it.

## Reporting a Vulnerability

1. Provide a clear summary of the issue.
2. Reproduction steps, including environment and affected versions.
3. Suggested mitigations if possible.
4. Contact information for follow-up.

We aim to respond within 48 hours and provide guidance for patching and disclosure.

## Responsible Disclosure

- Do not exploit the vulnerability beyond what is necessary to demonstrate the issue.
- Coordinate disclosure timing with the maintainers.
